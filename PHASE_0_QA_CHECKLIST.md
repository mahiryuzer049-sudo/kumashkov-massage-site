# PHASE 0 — Visual & UX QA Checklist

Дата: 2026-02-12  
Использовать после каждой фазы исправлений.

## A. Визуальная цельность

- [ ] Между секциями нет видимых швов/полос.
- [ ] Фоновые переходы плавные, без резких стыков.
- [ ] У блоков есть визуальная иерархия (главное/вторичное).
- [ ] Каждый мини-блок имеет свою «фишку» и визуально не дублирует соседний.
- [ ] Стиль считывается как премиальный, а не шаблонный.

## B. Контент и удержание

- [ ] На каждом экране есть понятный смысловой крючок.
- [ ] Текст не перегружает: есть баланс текста и визуала.
- [ ] Блок логично ведет в следующий блок (читается воронка).
- [ ] Тон романтичный и мягкий, но профессиональный и этичный.
- [ ] Нет гендерных ограничений в формулировках.

## C. Навигация и сценарий

- [ ] В шапке есть `Блог`.
- [ ] Навигация отражает реальную структуру страницы.
- [ ] Якоря ведут точно в нужные разделы.
- [ ] Mobile-меню содержит те же ключевые входы.
- [ ] Sticky-бар не мешает чтению и не перекрывает CTA.

## D. Интерактивы

- [ ] Сканер визуально выглядит как фигура человека.
- [ ] Точки сканера кликаются и дают корректный popup.
- [ ] Popup закрывается по `Esc`, клику вне и кнопке закрытия.
- [ ] Клавиатурный фокус не теряется в интерактивах.
- [ ] CTA из сканера и других секций открывают корректные ссылки.

## E. Масштаб и адаптив

- [ ] На `100%` desktop читать комфортно.
- [ ] На `125%` desktop нет наложений и поломок сетки.
- [ ] На mobile нет обрезаний, наложений и слишком мелкого текста.
- [ ] Кнопки и интерактивы имеют комфортный размер для нажатия.

## F. Технические проверки

- [ ] `npm run lint`
- [ ] `npm run build`
- [ ] `npm run lint:html`
- [ ] `npm run perf:budget`
- [ ] `npm run check`
